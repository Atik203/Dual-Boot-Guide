Phase 3
💾 Create Ubuntu Bootable USB
Download Ubuntu 26.04 LTS and flash it to a USB drive. You need a USB drive of at least 8GB — this will be erased.
Download & Flash
1
Download Ubuntu 26.04 LTS ISO
~5.5GB ▶
  1. Go to: ubuntu.com/download/desktop
  2. Download Ubuntu 26.04 LTS (the "LTS" version — supported until 2031)
  3. File will be named like: ubuntu-26.04-desktop-amd64.iso (~5.5GB)
  4. Verify the SHA256 checksum after download:
PowerShell
# Verify ISO integrity (compare with SHA256SUMS from releases.ubuntu.com/26.04/)
Get-FileHash "C:\Users\YourName\Downloads\ubuntu-26.04-desktop-amd64.iso" -Algorithm SHA256
💡
Why 26.04 LTS? Supported until April 2031, latest system packages (GNOME 50, Linux Kernel 7.0), and excellent Wayland performance. Avoid non-LTS releases for a stable development stack.
2
Flash ISO to USB with Balena Etcher
USB ERASED ▶
  1. Download Balena Etcher: etcher.balena.io (free, portable .exe)
  2. Plug in your USB drive (≥8GB) — all data will be erased
  3. Open Etcher → Click "Flash from file"
  4. Select your ubuntu-26.04-desktop-amd64.iso
  5. Click "Select target" → select your USB drive (double-check the size!)
  6. Click "Flash!" → enter admin password if prompted
  7. Wait ~10–15 minutes → Etcher will validate automatically
  8. When done: "Flash Complete!" + green checkmark
⚠️
After flashing, Windows may say "USB drive needs to be formatted" — click Cancel/No. The USB is correctly formatted as Linux bootable and Windows can't read it. Don't format it.
💡
Alternative to Etcher: Rufus (also free). If using Rufus: Partition scheme = GPT, Target system = UEFI (non CSM), File system = FAT32.
Phase 4 — Install + BIOS
🔧 BIOS Setup + Ubuntu Installation
Configure BIOS for dual boot, then install Ubuntu on the 512GB M.2 drive. The most critical phase — follow each step exactly.
Part A — BIOS Configuration (Legion 5)
1
Enter BIOS on Lenovo Legion 5
EXACT KEYS ▶
  1. Fully shut down Windows (not restart)
  2. Press power button → immediately and repeatedly press F2 (BIOS) or F12 (boot menu)
  3. If Windows boots normally → shut down and try again faster
⚠️
If Fast Boot is ON in Windows: You may not have time to press F2. Fix first: Settings → System → Power → "Additional power settings" → "Choose what power buttons do" → uncheck "Turn on fast startup" → Save. Then shut down normally and try F2.
ℹ️
Alternative: Hold Shift while clicking Restart in Windows → "Use a device" → "UEFI Firmware Settings". This takes you directly to BIOS.

Secure Boot Options & Tradeoffs:
  • Enabled (Recommended): Mandated for Windows 11 compliance and kernel-level anti-cheat engines (Riot Vanguard for Valorant/League of Legends). Requires completing MOK (Machine Owner Key) enrollment in Phase 6 to sign and load the Nvidia proprietary modules.

Installation type — CRITICAL partition selection
READ FULLY ▶
🔴
This is where accidental data loss happens. Do NOT click "Erase disk and install Ubuntu" or you'll lose Windows. Select manual partitioning.
  1. On "Disk setup" screen → Select "Manual partitioning"
  2. Click Next/Continue

You'll see a partition table. Identify your 512GB drive:

  • Look for the drive with ~476GB total (shows as /dev/nvme0n1 or /dev/nvme1n1)
  • The 1TB Windows drive will have Windows partitions (EFI, Recovery, C:) — leave this one ALONE
⚠️
If the 512GB drive has existing partitions (from your E: drive), select each partition on that drive and click the "-" button to delete them. This completely erases the drive. Only do this to the 512GB drive.

2
Install NVIDIA driver for RTX 3060 (with Secure Boot MOK)
GPU SETUP ▶

Ubuntu 26.04 runs a Wayland-only GNOME session. The open-source Nouveau driver is unstable under Wayland for NVIDIA GPUs. You MUST install the proprietary NVIDIA driver for basic performance and display correctness.

Terminal
# Check which NVIDIA driver is recommended
ubuntu-drivers devices

# Install the recommended driver (usually nvidia-driver-550 or 560)
sudo ubuntu-drivers autoinstall

# OR install specific version manually
sudo apt install -y nvidia-driver-560
⚠️
Secure Boot Password (MOK): During driver installation, a terminal prompt will pop up asking to configure a password for Secure Boot / MOK. Create a simple password (e.g., 12345678) and confirm it. You will need this key upon restart.

After installation completes, reboot the system:

Terminal
sudo reboot
🔵
CRITICAL: Enroll MOK on Reboot (Blue Screen)
  1. On boot, a blue screen will display: "Perform MOK management".
  2. Press any key within 10 seconds to enter the menu. (If you miss it, the drivers won't load and you'll boot with a laggy UI).
  3. Select Enroll MOK → select Continue → select Yes.
  4. Enter the password you created during terminal installation (characters will not show).
  5. Select Reboot.

Once booted back into Ubuntu, verify the driver is active:

Terminal
# Should show RTX 3060 info and driver version
nvidia-smi

# Expected output: NVIDIA-SMI 560.x | Driver 560.x | CUDA 12.x
⚠️
If nvidia-smi says "command not found" or shows errors, the signature block is preventing the driver from loading. Run sudo mokutil --import /var/lib/shim-signed/mok/MOK.der to re-trigger enrollment on the next reboot.

4
Configure KMS & DRM Mode Setting for Wayland Display Server
WAYLAND OPTIMIZE ▶

On hybrid Legion laptops running Ubuntu 26.04 (which is Wayland-only by default), you will experience micro-stutters, external monitor lag over HDMI, or severe gaming FPS drops in Steam/Proton (e.g. Cyberpunk 2077). This is caused by mismatched buffer synchronization between the iGPU (compositor) and dGPU (renderer). Enforcing Kernel Mode Setting (KMS) fixes this.

Terminal
# Edit the GRUB bootloader configuration
sudo nano /etc/default/grub

Find the line starting with GRUB_CMDLINE_LINUX_DEFAULT and append the DRM mode setting command inside the quotes. It should look like this:

/etc/default/grub
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nvidia-drm.modeset=1"

Save and exit Nano (Ctrl + O, Enter, Ctrl + X), then rebuild the bootloader config and reboot:

Terminal
# Update GRUB to load the DRM setting at boot
sudo update-grub

# Reboot the laptop
sudo reboot

After rebooting, verify DRM modeset is loaded: cat /sys/module/nvidia_drm/parameters/modeset (should output Y).
