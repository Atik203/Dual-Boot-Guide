"Windows Boot Manager" missing from GRUB after install
COMMON ▶

GRUB shows Ubuntu only, no Windows option.

Terminal (Ubuntu)
# Update GRUB — it should auto-detect Windows
sudo update-grub
# Look for: "Found Windows Boot Manager on /dev/nvme0n1p1"

# If still not found, install os-prober
sudo apt install os-prober

# Edit GRUB config to enable os-prober
sudo nano /etc/default/grub
# Add this line at the bottom:
# GRUB_DISABLE_OS_PROBER=false

# Save (Ctrl+X, Y, Enter) then:
sudo update-grub
sudo reboot
⚠
Windows time is wrong after switching from Ubuntu (Temporal Sync Bug)
COMMON ▶

Linux stores the hardware clock (RTC) in UTC format by default, whereas Windows 11 treats the hardware clock as local time. When dual-booting, this discrepancy results in a 6-hour offset (for Dhaka timezone, UTC+6) on every reboot.

🔴
Windows 11 24H2 Registry Bug: The standard registry modification (RealTimeIsUniversal) that forces Windows to use UTC fails to persist across hybrid shutdown and Fast Startup states in Windows 11 24H2. The registry key gets bypassed, resulting in recurring time drift.

The only permanent and robust solution is to configure Linux to use the local RTC standard instead, matching Windows' time system.

Terminal (Ubuntu) — Fix on Linux side
# Force Linux to store time in local RTC (aligning with Windows)
sudo timedatectl set-local-rtc 1 --adjust-system-clock

# Verify RTC status
timedatectl
ℹ️
Look for RTC in local TZ: yes in the output. A warning about local time will appear; this can be safely disregarded for dual-boot setups.
⚠
Black screen after Ubuntu boots (NVIDIA issue)
COMMON ▶

GRUB shows, Ubuntu starts loading, then black screen — nothing.

  1. At GRUB menu: press E to edit Ubuntu entry
  2. Find the line starting with linux
  3. At the end of that line, before quiet splash, add: nomodeset
  4. Press F10 to boot with this option
  5. Once in Ubuntu: install NVIDIA driver properly (Step 2 of Phase 6)
  6. After driver is installed and rebooted: remove nomodeset (it's only temporary)
ℹ️
To remove nomodeset permanently: sudo nano /etc/default/grub → find GRUB_CMDLINE_LINUX_DEFAULT → remove "nomodeset" → sudo update-grub

Hardware Controllers & Drivers (Wi-Fi, Audio, Touchpad)
✓
Intel AX200 160MHz — no issues expected
WORKS OOTB ▶

The Intel AX200 is fully supported by the mainline Linux kernel (iwlwifi driver) and works out of the box on Ubuntu 26.04. It supports 2.4/5/6GHz bands, Bluetooth 5.2, and 160MHz channel width. No manual firmware or driver installation needed.

If Wi-Fi doesn't appear after install, verify:

  1. Hardware switch is on — some Legion 5 models have an Fn+F8 (Airplane mode) toggle
  2. No RF kill: rfkill list — if blocked, run rfkill unblock wifi
  3. Driver loaded: lsmod | grep iwlwifi should show the module
⚠
Realtek ALC287 / ALC3306 Audio: Silent Internal Speakers
HARDWARE ▶

The Legion 5 features Realtek ALC287 or ALC3306 audio codecs paired with dual Cirrus Logic CS35L41 smart amplifiers connected via an I2S bus. While Linux successfully registers the ALC chip (enabling headphones and microphone), it fails to route sound signals through the secondary Cirrus Logic amplifier out of the box, resulting in dead silent speakers.

Fix Option 1: Install Cameron Berkenpas / Thiago Tei Firmware Patch (Recommended)

A community-maintained driver patch overrides the ACPI tables to correctly initialize the I2S routing block on boot.

Terminal
# Add community repository for Legion Audio Patches
sudo add-apt-repository ppa:cameron-berkenpas/legion-audio -y
sudo apt update
sudo apt install -y legion-audio-patch

# Reboot to apply the routing fix
sudo reboot

Fix Option 2: Suspend State Wake Hack (Zero-install Workaround)

If you prefer not to use a third-party PPA, a hardware initialization bypass exists. Suspending the computer and waking it forces the system BIOS to re-initialize the audio amplifier.

  • Close the laptop lid to suspend (enter S3 or modern standby sleep).
  • Wait 5 seconds until the power button LED begins pulsing slowly.
  • Open the lid and wake the computer. The speakers will now work until the next full shutdown.
⚠
I2C HID Touchpad Fails to Initialize or Register Deadlocks
HARDWARE ▶

The Legion's touchpad runs over an I2C HID bus. It occasionally stops working during warm restarts from Windows or when resuming from a deep sleep state because the I2C controller fails to clear its registers.

Resolution Steps:

  1. Kernel Upgrade (HWE): Upgrade to the latest Hardware Enablement (HWE) kernel, which includes patches for recovering from I2C bus timeouts and touchpad register resets:
Terminal
sudo apt update
sudo apt install --install-recommends linux-generic-hwe-26.04 -y
  1. Windows Device Manager Power Cycle: If the touchpad becomes unresponsive in both Windows and Linux, the touchpad's microcontroller has deadlocked. Boot into Windows → open Device Manager → expand Human Interface Devices → right-click I2C HID Device → click Disable → wait 3 seconds → click Enable. This forces a physical power-cycle code to the touchpad controller, clearing the lock state.

⚠
Micro-stutters, cursor lag, or window freezes under Wayland
WAYLAND/NVIDIA ▶

Since Ubuntu 26.04 is Wayland-only (X11 removed), hybrid GPU displays can experience minor refresh lag or screen freezes under Wayland.

  1. Verify you are indeed running on Wayland: echo $XDG_SESSION_TYPE (should return wayland).
  2. Ensure the proprietary NVIDIA driver is actually loaded: run nvidia-smi. If it doesn't load, Wayland runs on CPU/software rendering, causing massive lags.
  3. Fractional Scaling Stutter: If you use scale settings like 125% or 150%, Wayland + NVIDIA can stutter on hybrid setups. Try reverting display scale to 100% or 200%.
  4. GDM Configuration: Ensure GDM allows Wayland. Open /etc/gdm3/custom.conf and ensure WaylandEnable=false is fully commented out (prefixed with #).

Lenovo Legion Fan, Thermals, & Battery Conservation Management
LEGION SPECIFIC ▶

Standard Linux power tools fail to interface with Lenovo's proprietary EC (Embedded Controller) profiles. To safeguard battery health and manage thermals during high-compute AI training, configure the following settings:

1. Battery Conservation Mode (Limit charge to 60%)

To prevent battery degradation when the laptop is constantly plugged in, enable conservation mode:

Terminal (Immediate Effect)
# Enable conservation mode (limits maximum charge to 60%)
echo 1 | sudo tee /sys/bus/platform/drivers/ideapad_acpi/VPC2004:00/conservation_mode

To make this setting persistent across boots, integrate it into TLP:

Terminal
# Install TLP power management daemon
sudo apt install -y tlp
sudo systemctl enable tlp --now

# Write threshold configuration to TLP overrides
echo "STOP_CHARGE_THRESH_BAT0=60" | sudo tee /etc/tlp.d/01-battery-conservation-mode.conf

2. Custom Fan Curves & Power Profiles via LenovoLegionLinux

Standard fan control nodes are hidden on Legion laptops. Compiling this kernel driver exposes the thermal states and power switching profiles:

Terminal
# Install driver prerequisites
sudo apt install -y lm-sensors git build-essential dkms

# Clone driver source
git clone https://github.com/johnfanv2/LenovoLegionLinux.git ~/LenovoLegionLinux
cd ~/LenovoLegionLinux/kernel

# Compile and load the kernel module
sudo make install
sudo modprobe lenovo-legion-module

# Verify sysfs interface is present
ls /sys/kernel/legion/

You can now use `legion_gui` to draw custom fan curves. To switch power profiles directly via terminal (1 = Quiet, 2 = Balanced, 3 = Performance):

Terminal
echo 1 | sudo tee /sys/kernel/debug/legion/thermal_mode
💡
During heavy ML training runs, cap the RTX 3060 power draw to control temperatures: sudo nvidia-smi -pl 80 (limits GPU to 80W, reducing thermals with negligible performance loss).

Goodix Fingerprint Reader Biometric Configuration
BIOMETRICS ▶

The fingerprint reader integrated into the Legion 5's power button is a Goodix Touch OEM Device (TOD). It is not supported by default open-source drivers, but can be enabled using official proprietary wrappers:

Terminal
# Add Linux TOD repository
sudo add-apt-repository ppa:libfprint-tod1-group/ppa -y
sudo apt update

# Install fingerprint driver and PAM module
sudo apt install -y libfprint-2-tod1-goodix fprintd

# Restart fprintd service
sudo systemctl restart fprintd

# Enroll your fingerprint
fprintd-enroll

After successful enrollment, open GNOME Settings → Users → Fingerprint Login to enable biometric authentication for screen unlocking and sudo prompt elevation.

Phone not detected as drive letter in Windows
BACKUP ISSUE ▶

Vivo X300 Pro sometimes only charges and doesn't show as a drive.

  1. Pull down notification shade → Tap "Android System" USB notification
  2. Select "File Transfer" (not just charging)
  3. If no notification: Settings → Additional Settings → Developer Options → USB Configuration → select "File Transfer" (MTP)
  4. If Developer Options not visible: Settings → About Phone → tap "Build Number" 7 times
  5. Try a different USB cable — many cheap cables don't support data, only power
  6. Try USB 3.0 port on the Legion (blue ports)
💡
If MTP still fails: use AirDroid (free) or Snapdrop.net (peer-to-peer, no account) for wireless transfer over your home WiFi. Slower but reliable.
