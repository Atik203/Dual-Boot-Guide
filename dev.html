CUDA Toolkit Installation

Install CUDA 12.8 Toolkit
GPU COMPUTE ▶
⚠️
The CUDA Toolkit version should match what your driver supports. If nvidia-smi shows "CUDA Version: 12.8" or newer, install CUDA 12.8. Do not install CUDA 11.x.
Terminal
# Check max supported CUDA version from driver
nvidia-smi | grep "CUDA Version"
 
# Add NVIDIA CUDA repository for Ubuntu 26.04
wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2604/x86_64/cuda-keyring_1.1-1_all.deb
sudo dpkg -i cuda-keyring_1.1-1_all.deb
sudo apt-get update
 
# Install CUDA 12.8 (adjust version if needed)
sudo apt-get install -y cuda-toolkit-12-8
 
# Add CUDA to PATH — add these to ~/.bashrc
echo 'export PATH=/usr/local/cuda-12.8/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/cuda-12.8/lib64:$LD_LIBRARY_PATH' >> ~/.bashrc
source ~/.bashrc
 
# Verify CUDA installation
nvcc --version
# Should show: Cuda compilation tools, release 12.8

Install Miniconda (not Anaconda)
LIGHTER ▶

Miniconda is the same as Anaconda but without pre-installed packages — installs in ~500MB vs ~5GB.

Terminal
# Download Miniconda installer
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda.sh
 
# Verify download
sha256sum ~/miniconda.sh
 
# Install (in batch mode — no interactive prompts)
bash ~/miniconda.sh -b -p $HOME/miniconda3
 
# Initialize conda for bash
~/miniconda3/bin/conda init bash
source ~/.bashrc
 
# Verify
conda --version
 
# Disable auto-activate base env (good practice)
conda config --set auto_activate_base false
 
# Clean up installer
rm ~/miniconda.sh

LenovoLegionLinux Kernel Module — Fan & Thermal Tuning
LEGION THERMALS ▶

Running high-compute PyTorch models on the RTX 3060 will spike temperatures up to 90°C+. The generic Linux kernel cannot modify Legion Embedded Controller (EC) fan tables, causing default profiles to thermal-throttle. Compiling this driver enables custom fan curves and direct power profile switching (Quiet/Balanced/Performance).

Terminal
# Install dependencies for kernel compilation
sudo apt install -y lm-sensors git build-essential dkms
 
# Clone the Lenovo Legion Linux kernel driver repository
git clone https://github.com/johnfanv2/LenovoLegionLinux.git ~/LenovoLegionLinux
cd ~/LenovoLegionLinux
 
# Compile and insert the kernel module
sudo make install
sudo make reloadmodule
 
# Verify sensors are reading temperatures correctly
sensors

You can adjust power states (Quiet = 1, Balanced = 2, Performance = 3) directly from the terminal. Example for Quiet Mode to save thermals:

Terminal
# Echo thermal profiles directly to the EC debug node
echo 1 | sudo tee /sys/kernel/debug/legion/thermal_mode
 
# Set a GPU power cap at 80W during training (caps temps, minimal perf loss)
sudo nvidia-smi -pl 80

2
Install Docker + Docker Compose
CONTAINERS ▶
Terminal
# Install Docker Engine (official method, not snap)
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Add your user to docker group (no sudo needed for docker)
sudo usermod -aG docker $USER
newgrp docker

# Test Docker works
docker run hello-world
docker compose version
